# 🚀 جناح اختبارات الأداء المتكامل لـ Diwan Event (Locust Engine)

مرحبًا بك في الجناح الهندسي الكامل لاختبارات أداء منصة **ديوان لخدمات الفعاليات الكبرى**.  
صُممت هذه الأداة لمحاكاة ضغط حقيقي يصل إلى **1,200 مستخدم نشط متزامن** واختبار مدى متانة ومرونة الخادم وقاعدة البيانات، مع التركيز على السيناريوهات الحرجة مثل بوابات الدخول السريعة صباحاً واستقبال البيانات اللحظية عبر الـ WebSockets.

---

## 🛠️ مكونات الجناح الهندسي (الملفات المتكاملة)

يتكون النظام من 8 ملفات رئيسية تم تصميمها وتوزيعها بشكل احترافي:

1. **`requirements.txt`**: يحتوي على الاعتماديات اللازمة (`locust`, `websocket-client`, `requests`).
2. **`thresholds.py`**: يحدد معايير القبول والرفض الفنية (مثل زمن الاستجابة المطلق ونسبة الخطأ القصوى لكل مسار).
3. **`conftest.py`**: يقوم بتهيئة قاعدة البيانات تلقائياً وإنشاء الفعالية التجريبية و 100 مشارك افتراضي وحساب للـ Scanner، وحفظ المعرفات في ملف الإعداد `test_config.json`.
4. **`locustfile.py`**: السيناريوهات الكاملة والمختلطة لمحاكاة سلوك المستخدمين الفعليين بنسب واقعية.
5. **`scenarios/checkin_spike.py`**: سيناريو محدد لاختبار الضغط المكثف لبوابات الدخول بمعدل 30 مسحة/دقيقة لكل جهاز.
6. **`scenarios/ws_dashboard.py`**: سيناريو محاكاة شاشات العرض الحية المتصلة بالـ WebSocket لاستقبال تحديثات الحضور الفورية.
7. **`run_tests.sh`**: السكريبت المؤتمت لتشغيل البيانات والاختبار في الوضع الصامت (Headless) وحفظ التقارير.
8. **`README.md`**: هذا الدليل التفصيلي للتشغيل والصيانة.

## 👥 نموذج المستخدمين والمحاكاة الفنية (Users Model)

تم توزيع المستخدمين المتزامنين الـ **1200** بناءً على سلوك الفعالية الحقيقي:

| فئة المستخدم | الوزن (النسبة) | دور المحاكاة الفني | السلوك والمهام |
| :--- | :--- | :--- | :--- |
| **جهاز المسح (Scanner)** | **60%** (720 جهاز) | مسح رموز الـ QR بمعدل سريع (ثانية إلى 4 ثوانٍ) | تسجيل الحضور الفعلي `PATCH /check-in` والبحث في القوائم |
| **المشارك العام (Attendee)** | **25%** (300 مستخدم) | تصفح البوابة الشخصية، التسجيل الذاتي، تصفح الرعاة والبرامج | استعراض الملف الشخصي `GET /me` وعرض الإعدادات وجدول الجلسات |
| **المسؤول (Admin)** | **10%** (120 مستخدم) | مراقبة الإحصائيات الحية وسحب التقارير التفصيلية | جلب التحليلات `GET /summary` والإحصائيات المعقدة لـ `peak-hours` |
| **نظام المراقبة (Health Check)** | **5%** (60 مستخدم) | فحص سلامة النظام Readiness Probe بشكل دوري | استدعاء `/health` للتأكد من استقرار الخادم وقاعدة البيانات |

## 🚀 طريقة التشغيل والتهيئة (Quick Start)

### 1️⃣ تثبيت الاعتماديات
تأكد من تفعيل البيئة الافتراضية الخاصة بك، ثم قم بتثبيت الاعتماديات اللازمة:
```bash
pip install -r requirements.txt
```

### 2️⃣ تهيئة بيانات قاعدة البيانات تلقائياً
لتجنب أخطاء الفقدان (404) أو عدم الصلاحية (422)، قم بتشغيل سكريبت التهيئة الذكي الذي سيقوم بالتسجيل وإنشاء فعالية ومشاركين حقيقيين وحفظ بياناتهم في `test_config.json`:
```bash
python conftest.py
```

### 3️⃣ تشغيل لوحة التحكم التفاعلية (Locust Web UI)
لتشغيل واجهة المستخدم الرسومية الأنيقة ومراقبة الرسوم البيانية حياً:
```bash
locust -f locustfile.py
```
ثم افتح متصفحك على الرابط التالي: [http://localhost:8089](http://localhost:8089)

### 4️⃣ التشغيل التلقائي الكامل (Headless Execution)
لتشغيل الاختبار تلقائياً وتصدير التقارير في الوضع الصامت عبر سكريبت التشغيل المكتوب خصيصاً:
```bash
bash run_tests.sh
```

## 📈 معايير قبول الأداء (Performance Thresholds)

تم ضبط معايير صارمة للأداء لضمان جودة الاستخدام في الميدان:

* **التحضير وتسجيل الحضور (`PATCH /check-in`):**
  * يجب ألا يتجاوز زمن الاستجابة لـ **50%** من الطلبات **500ms**.
  * يجب ألا يتجاوز زمن الاستجابة لـ **95%** من الطلبات **1500ms**.
  * نسبة الأخطاء المسموحة أقل من **1%**.
* **التحليلات والمراقبة (`GET /summary`):**
  * متوسط زمن الاستجابة أقل من **800ms**.
* **الاتصالات الحية (WebSocket Connections):**
  * يجب الحفاظ على اتصال مستمر وثابت بدون انقطاع مفاجئ تحت ضغط 1,200 مستخدم.

## 📁 المجلدات والتقارير المستخرجة
عند انتهاء الاختبار بنجاح، سيتم حفظ التقارير التفصيلية في مجلد `load_tests/reports/`:
* `diwan_perf_report.html`: تقرير تفاعلي كامل ومصور يحتوي على الرسوم البيانية للأداء.
* `diwan_perf_report_stats.csv`: الإحصائيات الرقمية التفصيلية لكل مسار ونقطة نهاية.
* `diwan_perf_report_failures.csv`: قائمة شاملة ومحللة بأي أخطاء حدثت أثناء الضغط لدراسة مكامن الخلل.
