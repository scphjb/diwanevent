## ⚠️ الجزء الثاني: ما هو موجود هيكلياً لكن غير مفعّل (Scaffolded but not wired)

> [!WARNING]
> هذه الميزات لها ملفات وكود، لكنها **لا تعمل فعلياً** في بيئة الإنتاج.

### 1. 🔴 بوابة الدفع الإلكتروني (`payments.py`)
- **الحالة:** هيكل فقط (Scaffold) — لا يوجد ربط حقيقي بأي بوابة دفع
- **ما هو موجود:** Class `PaymentGateway` مع دوال `create_checkout_session` و `verify_payment`
- **المشكلة:** كود Stripe معلّق بالكامل (commented out). البوابة تعيد Mock URL فقط
- **صفحة الدفع التجريبية:** `/mock-payment` تعمل لكنها ليست بوابة حقيقية
- **ما يجب فعله:**
  - ربط مع Chargily Pay (أفضل بوابة جزائرية تدعم CIB/Edahabia)
  - أو ربط مع Stripe للدفع الدولي
  - إضافة Webhook حقيقي لتأكيد الدفع

### 2. 🔴 نظام الإشعارات بالبريد (`notifications.py`)
- **الحالة:** الكود مكتمل تقنياً لكن **معطّل**
- **المشكلة:** متغير `SMTP_PASSWORD` فارغ في `.env` → الدالة تتوقف فوراً (`return False`)
- **ما يجب فعله:**
  - إعداد حساب SMTP حقيقي (Gmail App Password أو Mailgun أو Sendinblue)
  - إضافة المتغيرات (`SMTP_SERVER`, `SMTP_PORT`, `SMTP_USERNAME`, `SMTP_PASSWORD`) في `.env`
  - تعديل رابط التذكرة من `localhost:8000` إلى الدومين الحقيقي

### 3. 🔴 قاعدة البيانات PostgreSQL (`database_v2.py` + `models.py`)
- **الحالة:** الهيكل جاهز لكن **غير مستخدم فعلياً**
- **المشكلة الجوهرية:** ملف `main.py` يستورد من `database.py` (SQLite القديم) وليس من `database_v2.py` (SQLAlchemy)
  ```python
  # main.py السطر 22:
  from database import init_db, get_db_connection, ...  # ← SQLite القديم!
  ```
- **النتيجة:** المنصة تعمل حالياً على **SQLite فقط** رغم وجود بنية PostgreSQL كاملة
- **ما يجب فعله:**
  - إعادة كتابة دوال `database.py` لتستخدم SQLAlchemy بدل SQLite المباشر
  - أو تعديل `main.py` ليستورد من `database_v2.py`
  - سكريبت الترحيل `migrate_to_pg.py` جاهز لكن لم يُختبر

### 4. 🟡 نظام Multi-Tenant (`models.py`)
- **الحالة:** النماذج (Models) تدعم Multi-Tenant نظرياً
- **المشكلة:** `main.py` لا يفلتر حسب `organization_id` أو `event_id`
- كل الاستعلامات تعمل على جدول واحد بدون فصل بين الأحداث/المؤسسات

### 5. 🟡 الاختبارات الآلية (Tests)
- **الحالة:** هيكل CI/CD موجود في `deploy.yml`
- **المشكلة:** الاختبارات معلّقة (`echo "Tests passed (Mock)"` السطر 26)
- لا يوجد مجلد `tests/` ولا ملفات pytest فعلية

### 6. 🟡 التحليلات المتقدمة (`analytics.py`)
- **الحالة:** الدوال موجودة لكن تعتمد على `get_db_connection()` بدون import
- **المشكلة:** ملف `analytics.py` لا يستورد `get_db_connection` → سيخطئ عند التشغيل
- **ما يجب فعله:** إضافة `from database import get_db_connection` في أعلى الملف
