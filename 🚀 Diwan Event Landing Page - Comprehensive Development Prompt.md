## 🎨 Design System & Visual Guidelines

### Color Palette
- **Primary Brand Color:** Teal/Turquoise (#1DB584 or #4ECDC4)
- **Secondary Color:** Deep Navy (#0F1419 or #1A1F2E)
- **Accent Color:** Gold/Amber (#D4AF37 for premium features)
- **Background:** White (#FFFFFF) or Off-white (#F8F9FA)
- **Text Primary:** Dark Gray (#1A1F2E)
- **Text Secondary:** Medium Gray (#6B7280)
- **Success:** Green (#10B981)
- **Warning:** Orange (#F59E0B)
- **Error:** Red (#EF4444)

### Typography
- **Headlines (H1-H3):** Inter, Poppins, or Roboto (Bold, 32px-64px)
- **Subheadings (H4-H5):** Inter, Poppins (Semibold, 18px-24px)
- **Body Text:** Inter, Open Sans (Regular, 14px-16px)
- **Small Text:** Inter (Regular, 12px-14px)

### Spacing & Layout
- **Container Max-Width:** 1200px
- **Padding:** 16px, 24px, 32px, 48px, 64px (scale)
- **Gap Between Sections:** 80px-120px
- **Card Padding:** 24px-32px

### Visual Effects
- **Shadows:** Subtle (0 4px 12px rgba(0,0,0,0.1))
- **Border Radius:** 8px-16px for cards, 4px for buttons
- **Transitions:** 300ms ease-in-out
- **Hover Effects:** Slight scale (1.02-1.05), shadow increase
- **Glassmorphism:** Semi-transparent backgrounds with backdrop blur for CTAs

### Icons
- Use Lucide React icons or Heroicons
- Size: 24px-48px depending on context
- Color: Match text color or use brand teal

---

## 📱 Responsive Design Requirements

**Breakpoints:**
- Mobile: 320px - 640px
- Tablet: 641px - 1024px
- Desktop: 1025px+

**Mobile Considerations:**
- Stack all multi-column sections vertically
- Reduce font sizes by 10-15%
- Increase touch target sizes (min 44px)
- Simplify navigation (hamburger menu)
- Full-width cards with padding
- Single-column testimonial carousel

---

## 🌐 Multilingual Support (i18n)

**Supported Languages:**
- Arabic (AR) - RTL layout
- English (EN) - LTR
- French (FR) - LTR
- Spanish (ES) - LTR

**Implementation:**
- Use i18n library (next-i18n-router or similar)
- Language selector in header and footer
- All text strings in translation files
- Date/number formatting per locale
- RTL CSS for Arabic

---

## ♿ Accessibility Requirements

- WCAG 2.1 AA compliance
- Semantic HTML (proper heading hierarchy, alt text for images)
- Keyboard navigation support
- Focus indicators on all interactive elements
- Color contrast ratio ≥ 4.5:1 for text
- ARIA labels for screen readers
- Skip navigation link

## 🔧 Technical Implementation

### Tech Stack
- **Framework:** React 19 with TypeScript
- **Styling:** Tailwind CSS 4
- **Components:** shadcn/ui
- **Animations:** Framer Motion
- **Forms:** React Hook Form + Zod validation
- **HTTP Client:** Fetch API or Axios
- **Build Tool:** Vite
- **Deployment:** Vercel, Netlify, or custom

### Key Features to Implement
- Smooth scroll behavior
- Lazy loading for images
- Intersection Observer for animations
- Form validation with error messages
- Newsletter signup integration
- Demo request form with email validation
- Analytics tracking (Google Analytics, Mixpanel)
- SEO optimization (meta tags, structured data)

### Performance Targets
- Lighthouse Score: ≥ 90
- First Contentful Paint (FCP): < 1.5s
- Largest Contentful Paint (LCP): < 2.5s
- Cumulative Layout Shift (CLS): < 0.1
